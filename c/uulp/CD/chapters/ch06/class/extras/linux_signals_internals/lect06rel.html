Lecture 6 Signals
Signals are sent to processes. Requests for signals come from users, from the kernel, and from other processes. The result of delivering a signal to a process can be determined by the receiving process. Therefore, there are four things to look at:
  • The structure of a process (called a task on Linux)
  • The request for a signal
  • The delivery of a signal
  • The response to a signal

The Linux source code has all the details of these components

  • signal.h The signal() and sigaction() system calls use the values and structures in this file.

  • kernel/signal.c The code for the signal() and sigaction() system calls are in this file. The function called kill_something() is the general interface that interprets the arguments to kill(). The function called do_sigaction() examines and acts on the values in the struct sigaction. You can see in this code what happens to the values you pass to the kernel.

  • sched.h The receiver of a signal is a task, defined in this file. Look for the definition of struct task_struct to see all the attributes of a process. Look for the signal-related members.

  • exit.c The mechanism for terminating a process, notifying its relatives, and waiting for children is coded in this file. You can see here who is notified and how.
