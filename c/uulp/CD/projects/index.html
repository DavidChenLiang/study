Programming Projects: Homework Assignments
This section contains programming projects. These assignments include the description you give to the students and supporting documents and code they use to do the work.
README Ideas on using programming projects
aac
pdf,  txt
dir		 User login accounting (chapter 2)
This project is to write a version of the Unix ac utility. It involves reading the wtmp file, working with the ideas of users, tty lines, login times. A complete version requires using linked lists.
llast
pdf,  txt
dir		 A version of last (chapter 2)
This project is to write a version of the Unix last utility. It involves reading the wtmp file, working with the ideas of users, tty lines, login times. A complete version requires using linked lists. This project requires reading a file backwards, thereby introducing some subtle, useful problems. This one is trickier than aac.
watch
pdf,  txt
dir		 Monitor user logins/logouts (chapter 2)
This project is to write a program that implements the watch feature of the tcsh. The program is to report when specified users login or logout of the system. It involves reading the utmp file, working with the ideas of users, tty lines, login times. It introduces the use of stat() and the idea of background processes.
pfind
pdf,  txt
dir		 A simplfied version of find (chapters 3 and 4)
This project is to write a version of the Unix find utility. It involves reading directories, learning about file types, using stat and lstat. A good application of recursion.
dulite
pdf,  txt
dir		 A version of the du (chapters 3 and 4)
This project is to write a version of the Unix du utility. It involves reading directories, learning about file types, using stat and lstat, understanding disk block allocation for files, and some of the members of the stat struct. A very nice application of recursion; it requires accumulating totals from subtrees. This one is trickier than pfind.
lsr1ff
pdf,  txt
dir		 Recursive directory lister (chapters 3 and 4)
This project is to write a version of the Unix ls utility. This project requires writing a version of ls that recursively lists all items in a directory tree. It involves reading directories, learning about all file types, using stat and lstat, understanding symbolic links, file sizes. A good application of recursion.
sttyl
pdf,  txt
dir		 Terminal Driver Control (chapters 5 and 6)
This project is to write a version of the Unix stty utility. It involves understanding the role and details of a device driver, the struct termios type, the ideas of control characters. The assignment requires building software that is table-driven.
pong
pdf,  txt
dir		 Action video game (chapter 7)
This project is to write a curses based pong game. Most students like this one a lot; finally they can show their course work to friends and family. More than the fun value, the project requires using curses, timers, signals, and it gets students programming and thinking about concurrent activities. The ideas in this chapter prepare students to learn about multithreaded programs. It also invites object oriented design.
smsh
pdf,  txt
dir		 A programmable shell (chapters 8 and 9)
This project is to extend the functions of the small shell presented in the text. The core of a Unix course, writing a shell requires understanding fork, exec, wait, the environment, and signals. This assignment also requires students to learn about writing an interpreter. Good students can write a version that builds parse trees and then passes them to an execution unit. This assignment has something for everyone.
netpong
pdf,  txt
dir		 A network video game (chapters 10-12)
This is a great project. Students convert their pong game into a two-player version using TCP/IP sockets. They have to code to an RFC spec, use sockets, curses, select, timers. Adventurous students can replace the curses interface with an X11 version.
wsng
pdf,  txt
dir		 Web Server (chapters 10-12)
This project is to extend the simple web server presented in the text. It provides practice programming with sockets, using a protocol, using fork, exec, I/O redirection, environment variables, directories, and files.
sumac
pdf,  txt
dir		 A pop3 client (chapters 10-12)
This project is to write an Internet mail client. The project requires reading and understanding an RFC, using sockets, files, curses, and processes. Students are often amazed to learn exactly how reading email works.
