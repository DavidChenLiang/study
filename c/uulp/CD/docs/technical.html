How I Teach with
Understanding Unix/Linux Programming
Understanding Unix/Linux Programming and all the material in this package are based on my course called Unix Systems Programming. The course follows a pretty standard model.

Classes
I give a two-hour class each week. I have a Unix system and a screen projector so everyone can see what I type and run on the system. I demonstrate programs, explain how Unix works, write code, and discuss the ideas and techniques.

The lecture material consist of:

  • Class Outline - the agenda
  • Sample Code - discuss code, run programs
  • Blackboard Images (as slides)

This Instructor's Resource package includes outlines, code, and images for all fifteen chapters. You can print out the handouts, copy the source code to your machine, show the slides, and explain how things work.

Using Unix during Classes

Having a live system during class allows me to answer questions by demonstrating, rather than stating, the answers. If a student asks "What would happen if you changed that line to read..", I can make the change to the code, compile it, and see what happens. By using Unix to teach, I try to encourage students to try similar experiments with sample code outside of class.

Projects
I assign six programming projects during the term. Students have two to three weeks to do each one. Each assignment is a complete Unix program, based on ideas and code from class and reading, designed to provide practice with the material and also to offer chances for more exploration and discovery.

This package has twelve projects. You can use the projects as they are. Simply add your course name and change some directory paths in the document, then print them out. These are not simple projects, but students find them interesting and worth the work.

Discussions
I meet with students, and teaching assistants conduct discussion sessions. During these meetings, we usually focus on the programming projects. I use the project to emphasize certain elements of how Unix works and what the API offers. I ask students about their design ideas for their solutions, and I use the time to discuss software design, data structures, and algorithms.

Exams
I give a final exam. This package has all twelve final exams I gave. You can use them as models, cut and paste from them, or use them as they are. Other people have copies of them, so they are not completely secure.

Overall
The course has worked well in this format all its twelve years. The clean, sensible design of Unix makes sense to students, and the approach of studing real programs and writing real programs keeps people motivated.
